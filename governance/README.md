# Governance

IKP-Labs governance defines **why** this project exists, **what values** guide its decisions, and **what standards** all contributors must follow.

## The 6-Layer Model

Governance in IKP-Labs follows a 6-layer hierarchy. Higher layers provide context for lower ones — when a lower layer is ambiguous, look up to the next layer for guidance.

```
Layer 1  governance/vision.md           Purpose — why this project exists
Layer 2  governance/principles.md       Values — what guides decisions
Layer 3  governance/conventions.md      Standards — naming, format, structure
Layer 4  .workflow-template.md          Development — task types, commit strategy
Layer 5  .workflow-template.md          Workflows — step-by-step procedures
Layer 6  plans/                         Plans — individual work tracking
```

For a full explanation of the model and how layers interact, see [`docs/explanation/governance.md`](../docs/explanation/governance.md).
