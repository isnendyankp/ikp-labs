## Posting Best Practices

### Optimal Posting Time
- **Best days:** Tuesday, Wednesday (highest professional engagement)
- **Best time:** 9:00 AM - 10:00 AM (before work rush)
- **Avoid:** Mondays (inbox overload), Fridays (weekend mindset)

### Engagement Strategy

**1. Post Timing:**
- Schedule for Tuesday/Wednesday morning
- Allow 24-48 hours for organic reach

**2. First Comment:**
- Post a follow-up comment 5-10 minutes after publishing
- Ask a technical question or share additional context
- Example: "What's been your experience with [technology]? Any tips for [specific challenge]?"

**3. Hashtag Strategy:**
- Use 5-10 hashtags (not more)
- First 3 hashtags get most visibility
- Order: Primary tech â†’ Secondary tech â†’ Practice/methodology

**4. Image/Screenshot Suggestions:**
- Code snippets (Before/After)
- Test results dashboard
- Metrics visualization
- Architecture diagrams
- Terminal output (test pass summary)

**5. Engagement Monitoring:**
- Check first 2 hours for comments
- Respond to all comments within 24 hours
- Like and reply to meaningful engagement

### Content Checklist

Before posting, verify:
- [ ] Character count: 900-1,400 (or 300-600 for short version)
- [ ] Clear hook in first line
- [ ] Metrics/results are quantifiable
- [ ] Tech stack mentioned
- [ ] Repository link included
- [ ] 5-10 relevant hashtags
- [ ] Line breaks for visual scanning
- [ ] Emojis used professionally (not excessive)
- [ ] No typos or grammar errors
- [ ] Call-to-action or discussion prompt included
- [ ] Screenshot/code image ready (if applicable)

## Weekly Posting Workflow

### 1. Complete Weekly Milestone
- Finish feature/testing/refactoring
- Document results (metrics, learnings)
- Commit and push to GitHub

### 2. Draft Post
- Open this template
- Fill in each section
- Use examples as reference
- Check character count

### 3. Review & Edit
- Read aloud for flow
- Verify metrics are accurate
- Check links work
- Run through content checklist

### 4. Prepare Supporting Materials
- Take screenshots
- Create code snippet images
- Export test results
- Prepare architecture diagrams (if needed)

### 5. Schedule & Post
- Post on Tuesday/Wednesday 9-10 AM
- Add first comment within 10 minutes
- Monitor engagement first 2 hours

### 6. Track Results
- Update History file with engagement metrics
- Note what worked well
- Adjust strategy for next post

### 7. Archive
- Add full post to [linkedin-posts-history.md](../History/linkedin-posts-history.md)
- Include metadata (date, topic, engagement)
- Link to commit/milestone in GitHub
